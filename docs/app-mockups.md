# AirSense Mobile App - UI Mockups

## Design System

### Color Palette
- **Primary:** #2196F3 (Blue)
- **Background:** #121212 (Dark) / #FFFFFF (Light)
- **Surface:** #1E1E1E (Dark) / #F5F5F5 (Light)
- **Success/Green:** #4CAF50
- **Warning/Yellow:** #FFC107
- **Danger/Orange:** #FF9800
- **Critical/Red:** #F44336
- **Text Primary:** #FFFFFF (Dark) / #000000 (Light)
- **Text Secondary:** #B0B0B0 (Dark) / #666666 (Light)

### Typography
- **Headings:** Inter/SF Pro Display (Bold)
- **Body:** Inter/SF Pro Text (Regular)
- **Numbers:** SF Mono/Roboto Mono (for sensor readings)

## Accessibility Features

- **High Contrast Mode:** Enhanced color differences
- **Large Text Support:** Respects system font scaling
- **Screen Reader:** Full VoiceOver/TalkBack support
- **Haptic Feedback:** For critical alerts
- **Color Blind Mode:** Icon + text indicators (not color alone)

---

## Technical Notes

### State Management
- Use Redux or Zustand for app state
- Persist connection state and settings locally
- Cache last 24h of data in AsyncStorage

### BLE Communication
- Subscribe to all characteristics on connect
- Handle reconnection automatically
- Show connection quality indicator
- Timeout after 30s of no data

### Performance
- Debounce chart updates (max 1 update/second)
- Lazy load history data
- Use React.memo for sensor cards
- Optimize chart rendering with react-native-chart-kit
